1. 背景与目标

Cycle 1 / 2 落地的两个 V10 Decider 策略（faber-gtaa / donchian-200）都是每指数独立 in/out的逻辑——每个指数自己的状态机决定自己持不持仓。组合层是简单等权（INDEX_CAPITAL 起步 × 27 个指数 = 270k）。

Dual Momentum（Antonacci 2014）是横截面 top-K选股逻辑——每月 universe scan 全部 27 个指数，按 lookback 期收益排序，挑 top-5 等权持有，下月 rebalance。这与 in/out 逻辑完全不同：

  • in/out：决策粒度 = per-index per-bar（27 个独立判断）
  • cross-sectional：决策粒度 = portfolio per-rebalance（1 个 universe-wide 判断）

Cycle 3 目标：在 V10 框架下落地 cross-sectional-topk aggregator + Dual Momentum top-5 策略，与 cycle 1 / 2 的两个 in/out 策略形成"决策粒度对照实验"。最终 5 策略对比报告。

2. Dual Momentum 算法

每月 rebalance（月线 close 时点）：

  1. 对 universe 中每个指数 i 算 lookback 期收益： r_i = close_i[t] / close_i[t-L] - 1 （L = 12 月，从论文）
  2. 绝对动量过滤：仅保留 r_i ≥ abs_threshold（默认 0%）的指数。低于阈值的视为"无趋势"，不持仓。
  3. 相对动量排序：在合格指数中按 r_i 降序排，取 top-K（K=5）。
  4. 等权持有：当前资金平均分给 top-K 持仓直到下月 rebalance。
  5. 不合格 → 不换债券/现金（简化），持仓资金 idle 在 cash 池（不产生收益）。

Antonacci 论文用 K=1 但 universe 仅 4 个 ETF。本设计 universe 27 个，K=5 提供合理分散度。

3.4 NoOp Decider（占位）

cross-sectional 路径不走 per-bar decide 逻辑，但 Strategy Protocol 要求 decider: Decider。用 DualMomentumNoOpDecider 占位：

class DualMomentumNoOpDecider:
    """cross-sectional 策略占位 Decider。decide 永远返回 None。

    cross-sectional 决策走 _run_cross_sectional_topk 的 universe-wide scan，
    不调用 decide(*, cycle, bar, position_shares)。本类仅满足 Protocol 契约。
    """
    name = "dual-momentum-noop"
    required_indicators: Tuple[Tuple[str, str, int], ...] = ()

    def decide(self, *, cycle, bar, position_shares):
        return None

5. _run_cross_sectional_topk 流程

  1. 校验 strategy.cycles == ("M",)（cross-sectional 仅月度 rebalance）
  2. 从 strategy.params 读 lookback_months / topk / abs_threshold
  3. load 所有指数 monthly close 序列
  4. 调 cross_sectional.build_holdings_schedule(monthly_close_by_code, lookback, topk, abs_threshold) 返回 Dict[Timestamp, Set[code]]（每月 rebalance 时的 holdings 集合）
  5. 对每个窗口 N 调 run_portfolio_window_cross_sectional_topk 算 portfolio equity_curve + WindowResult
  6. 返回 (strategy, registry, index_data, full_results={}, window_results)——full_results 为空 dict，告知 compare_report"该策略无 per-index 数据"

6. run_portfolio_window_cross_sectional_topk 设计

  1. 窗口起点 = as_of − N 年
  2. 初始资金 = TOTAL_CAPITAL（= 27 × INDEX_CAPITAL = 270k，与 equal-weight 等价）
  3. 从 holdings_schedule 取窗口内的 rebalance dates
  4. 对每个月 rebalance：
    • 当前总资金 = 上月末资金（from equity_curve）
    • 本月 holdings = schedule[date]
    • 分给 holdings 后，下月按各持仓的 close 涨跌算月末资金
    • portfolio_return_t = mean of holdings_{t-1} returns from close[t-1] to close[t]
  5. 累积 equity_curve（月度 series）→ 算 CAGR + MDD
  6. WindowResult 的 per_index = []（横截面无 per-index 概念）

write_compare_report 在生成分指数差异表时，目前迭代 registry 取 other_full[meta.code]。如果 other_full 为空 dict（cross-sectional 策略），diff 列表为空。

11. 关键不变量

  • Look-ahead 严禁：lookback return 必须用 close[t] vs close[t-L]，t 是当前 rebalance 月；t+1 月的 portfolio_return 用 t 月末的 holdings × (close[t+1] / close[t] - 1)
  • 资金守恒：所有月 portfolio equity 之和应等于初始资金 × 累积 portfolio return；不能凭空产生或丢失资金
  • 持仓数 ≤ K：select_topk 严格保证 |holdings| ≤ K
  • 合格性早过滤：abs_threshold 在排序前过滤；若所有指数都 < threshold，holdings = 空 set，该月 equity 不变（cash idle）
  • per-cycle 独立：本策略仅跑 M cycle，cycles=("M",) 强制约束

12. 风险与权衡

风险缓解
cross-sectional 路径不复用 V10 Decider 框架，引入新 code path新模块 cross_sectional.py 独立 + 单测覆盖核心逻辑；run.py / window_engine.py 改动最小
Strategy.params 用 Dict[str, Any]，类型弱、易写错 keycross_sectional.py 入口处显式从 params 读字段、给默认值；测试覆盖默认 + 自定义
compare_report 对横截面策略的分指数表语义不同（in/out 是 per-index 持仓表现，cross-sectional 该指数仅在被选中期持有）跳过 cross-sectional 的分指数表，加文字说明；不强行编造 per-index 数据
K=5 在 27 universe 上是否合理（≈ 18.5%）需实测验证注册新策略名 dual-momentum-top3 / dual-momentum-top7 即可扫参，框架天然支持
不合格时 cash idle，bear market 全程 0 收益（not even 债券对冲）设计选择（保持简洁）；后续可扩展 fallback_to_cash → fallback_to_bond，但需要新数据源

13. 不在范围

  • 动态 K 调整（如根据 volatility regime 改变 K）— 留 cycle 5+
