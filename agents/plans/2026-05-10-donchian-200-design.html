1. 背景与目标

Cycle 1 落地了：

  • 已有 combined-27 universe（27 个唯一指数：A 股 19 + 港股 3 + 美股 2 + 加密 1 + 商品 2）
  • 已有 equal-weight aggregator（单 cycle / 每指数等权 / 不调 Calmar）
  • 已有 N≥2 策略 compare 报告
  • 已有 Faber GTAA（月线 MA10 趋势跟踪，cycle 1 实测 4 窗口全输 baseline）

Cycle 2 目标：在同一个 equal-weight 框架下落地 Donchian 通道突破策略，与 Faber 同 timeframe（月线）做"价格突破" vs "均值穿越"的对照实验。

核心命题：

  • Faber 的 close vs MA10 是均值法——平滑、对噪声不敏感
  • Donchian 的 close vs max/min 是极值法——对突破更严格、入出场更慢
  • 同 universe / 同 timeframe / 同 aggregator 下，二者孰优？

本周期不做 dispatch / aggregator / portfolio 层任何改动——cycle 1 的框架已就绪，cycle 2 仅需新增一个 Decider + 注册一个策略名。

2. DonchianBreakoutDecider 规格

2.1 算法

每根月线 K 线（cycle="M"）：

  1. 读 bar.close，若 NaN → 返回 None
  2. 从 per-cycle buffer 取过去 N 个月的 close 序列（不含当月）
  3. 若空仓且 buffer 长度 ≥ entry_window：
    • entry_high = max(buffer[-entry_window:])
    • 若 close > entry_high（严格大于）→ BUY 信号
  4. 否则若持仓且 buffer 长度 ≥ exit_window：
    • exit_low = min(buffer[-exit_window:])
    • 若 close < exit_low（严格小于）→ SELL 信号
  5. 更新 buffer：append 当月 close；裁剪到最近 max(entry_window, exit_window) 个
  6. 返回信号（或 None）

2.2 与 Faber 的对照

维度Faber GTAA（cycle 1）Donchian 200（cycle 2）
判别式close > MA10（10 月均值）close > max(过去 10 月 close)
出场条件close ≤ MA10（同位均值）close < min(过去 5 月 close)
状态UP/DOWN（每月 0/1 翻转）HOLD/OUT（仅在突破/跌破时翻）
入场频率较高（均值穿越易触发）较低（必须创 N 月新高）
出场速度较快（close ≤ MA 立即翻 DOWN）较慢（必须破 N/2 月新低）
required_indicators(("M", "ma10", 10),)() 自维护 buffer

2.3 默认参数

  • entry_window = 10（10 个月新高 ≈ 200 个交易日新高）
  • exit_window = 5（5 个月新低，half-N，海龟系统经典出场）

未来扫参：直接注册新策略名 donchian-12-6 / donchian-8-4 等，框架支持。

2.4 状态隔离

Decider 实例内部用 self._close_buffer_by_cycle: Dict[str, List[float]] 按 cycle key 隔离 buffer。equal-weight 路径下每个指数都构造 fresh decider 实例（参考 cycle 1 _run_equal_weight 直接传 strategy；engine 内 cycles=("M",) 时只跑该 cycle），所以 buffer 不会跨指数串数据。

3. donchian-200 策略注册

@register("donchian-200")
def _donchian_200() -> Strategy:
    return Strategy(
        name="donchian-200",
        decider=DonchianBreakoutDecider(entry_window=10, exit_window=5),
        filters=(),
        cycles=("M",),
        aggregator="equal-weight",
    )

命名说明："200" 指日线级别 ≈ 200 交易日（与海龟系统 1 / Donchian 200 文献命名一致）；月线实现用 10/5 是同 timeframe 等价。

4. equal-weight 流程复用

Cycle 1 已落地 _run_equal_weight + run_portfolio_window_equal_weight，本周期零改动。Donchian 走流程：

  1. run.py --strategy donchian-200 --universe combined-27 --windows N
  2. _run_one_strategy dispatch → strategy.aggregator == "equal-weight" → _run_equal_weight
  3. 每指数 fresh DonchianBreakoutDecider 实例，run_with_strategy 跑 monthly cycle 全历史
  4. run_portfolio_window_equal_weight 在 N 年窗口内重跑 + 等权聚合
  5. WindowResult 出 CAGR / MDD

因 required_indicators=()，_ensure_indicators 是 no-op，不会在 IndexData 上加任何列。

8. 关键不变量

  • 状态机正确性：HOLD/OUT 状态必须严格响应 close 与 entry_high / exit_low 的关系——not loose threshold（用 > 和 <，不是 >= 和 <=）
  • Buffer 时序：决策必须基于"过去 N 月 close"（不含当月），更新发生在决策之后——避免 look-ahead bias
  • Per-cycle 隔离：buffer 按 cycle key 分桶；本策略只跑 M cycle，但 buffer 设计支持多 cycle（未来 D/W donchian 同框架可加）
  • 每指数 fresh decider：equal-weight 路径下，每指数 run_with_strategy 时直接传 strategy（cycles=("M",)），decider 内部 buffer 不会跨指数污染——这点 cycle 1 _run_equal_weight 已建立

9. 风险与权衡

风险缓解
Donchian 入场严格（必须创 N 月新高），bear market 全程不入场，CAGR 跑输 baseline这是策略本身特性。报告中对比 faber 的"易入场易出场" vs donchian 的"难入场难出场"两种风格
buffer 在新指数（如北证50 2022 起步）冷启动期长达 10 个月，错过早期机会预期行为；与 Faber MA10 在新指数上启动期失真同样问题
同时多个指数都"创 N 月新高"，equal-weight 等权后某些 universe 可能集中在 1-2 类指数（如某段牛市同时只有大盘 ETF 突破）本策略不做横截面 top-K（那是 cycle 3 Dual Momentum 的事）；equal-weight 框架接受"同时多空仓位"作为正常状态
每指数 fresh decider buffer 在第一次 run_with_strategy 调用时全历史 warm up，第二次 run_portfolio_window_equal_weight 重跑窗口时又会重新 warm up与 Faber 同问题；cycle 1 已验证 equal-weight 等权聚合的窗口重跑机制工作正确
